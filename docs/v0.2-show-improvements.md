## Issues to Fix

### 1. Box-drawing characters
Current pipe-delimited table is hard to parse. Use Unicode box-drawing (в”Җ в”Ӯ в”Ң в”җ в”” в”ҳ в”ң в”Ө в”¬ в”ҙ в”ј) for proper grid rendering.

### 2. Split halves
Left and right halves shown inline with `|    |` separator. Should render as two separate grids, labeled "Left:" and "Right:", matching the physical keyboard split.

### 3. Thumb cluster geometry
Thumbs shown as regular keys in a flat row. Should render with:
- **Small key** (inner thumb): standard width, offset toward the matrix
- **Large key** (outer thumb): wider, offset outward
- Visual distinction between the two sizes
- The Voyager thumb geometry: small key is closer to the matrix, large key is further out and wider

### 4. Truncated keycodes
`RALT(KвҖҰ` is unreadable. Full keycode names should be shown, with cell width expanding as needed. For very long keycodes, use human-friendly aliases:
- `RALT(KC_N)` вҶ’ `Гұ`
- `LALT_T(KC_I)` вҶ’ `I/Alt`
- `LCTL_T(KC_E)` вҶ’ `E/Ctl`
- `LSFT_T(KC_A)` вҶ’ `A/Sft`
- `LGUI_T(KC_R)` вҶ’ `R/Gui`
- `LT(Symbols, KC_ENT)` вҶ’ `ENT/Sym`
- `LT(Nav, KC_BSPC)` вҶ’ `BSP/Nav`
- `CK_FATARROW` вҶ’ `=>`
- `CK_DCOLN` вҶ’ `::`

### 5. Human-friendly labels
The current short codes (`LA:I`, `LC:E`, `LS:A`, `LG:R`) are QMK-internal notation. Use the format users think in:
- `LA:I` вҶ’ `I/Alt` (letter/modifier)
- `1:ENT` вҶ’ `ENT/Sym` (tap/layer)

### 6. Legend / key for notation
Add a footer legend explaining:
- `X/Mod` = tap for X, hold for Mod
- `X/Layer` = tap for X, hold to activate Layer
- Combo hints if applicable

### 7. Optional: finger zone coloring
Terminal color support (ANSI) to shade columns by finger assignment:
- Pinky columns: dim
- Ring: normal
- Middle: normal
- Index: normal
- Index inner stretch: slightly dim
- Outer columns: dim

Low priority вҖ” box-drawing and split rendering are the critical improvements.

## Implementation Notes

The renderer lives in `src/commands/show.rs` (or the display module). The Voyager geometry in `src/schema/geometry.rs` should provide thumb metadata (position, size) so the renderer can adapt per board.
