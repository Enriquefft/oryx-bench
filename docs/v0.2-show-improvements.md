## Issues to Fix

### 1. Box-drawing characters
Current pipe-delimited table is hard to parse. Use Unicode box-drawing (─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼) for proper grid rendering.

### 2. Split halves
Left and right halves shown inline with `|    |` separator. Should render as two separate grids, labeled "Left:" and "Right:", matching the physical keyboard split.

### 3. Thumb cluster geometry
Thumbs shown as regular keys in a flat row. Should render with:
- **Small key** (inner thumb): standard width, offset toward the matrix
- **Large key** (outer thumb): wider, offset outward
- Visual distinction between the two sizes
- The Voyager thumb geometry: small key is closer to the matrix, large key is further out and wider

### 4. Dynamic cell widths (no truncation)
Fixed 7-char cell width causes truncation (`LT(Sym…`). Use dynamic per-column widths — each column expands to fit its longest label. No ellipsis needed.

### 5. Human-friendly labels
The current compact format uses QMK-internal notation (`LA:ENT`, `1:BSPC`, `/LSFT`). Use the format users think in:

Mod-taps — tap/modifier (friendly name, no L/R since position implies hand):
- `LA:ENT` → `ENT/Alt`
- `LC:E` → `E/Ctl`
- `LS:A` → `A/Sft`

Layer-taps — tap/layer_name (actual layer name, not position number):
- `1:BSPC` → `BSPC/Sym+Num`
- `2:DEL` → `DEL/Brd+Sys`

Hold-only — /modifier (friendly):
- `/LSFT` → `/Sft`
- `/LCTL` → `/Ctl`

Punctuation keycodes — actual symbols instead of QMK short codes:
- `COMM` → `,`
- `SLSH` → `/`
- `QUOT` → `'`
- `SCLN` → `;`
- `MINS` → `-`
- `EQL` → `=`
- `BSLS` → `\`
- `GRV` → `` ` ``

### 6. Legend / key for notation
Add a footer legend explaining:
- `X/Mod` = tap for X, hold for Mod
- `X/Layer` = tap for X, hold to activate Layer
- Combo hints if applicable

### 7. Optional: finger zone coloring
Terminal color support (ANSI) to shade columns by finger assignment:
- Pinky columns: dim
- Ring: normal
- Middle: normal
- Index: normal
- Index inner stretch: slightly dim
- Outer columns: dim

Low priority — box-drawing and split rendering are the critical improvements.

## Implementation Notes

The show command handler lives in `src/commands/show.rs`, which delegates to the ASCII renderer at `src/render/ascii.rs`. The Voyager geometry is in `src/schema/geometry/voyager.rs` (trait + registry in `src/schema/geometry/mod.rs`). The `ThumbCluster` struct needs a `ThumbKeyWidth` (Standard/Wide) per key so the renderer can distinguish inner (1u) vs outer (1.5u) thumb keys.
